- 連結器（インターフェース）の価値を理解する

## 構成（約30分）

1. **車両はなぜバラバラなのか？** (5分)
   - なぜ1本の巨大な長い車両にしないのか？
   - カーブ曲がれない、1箇所壊れたら全滅、編成を変えられない

2. **連結器（インターフェース）の魔法** (10分)
   - 蒸気機関車でも電気機関車でも、客車を引っ張れる
   - 「連結器の形（Interface）」さえ合っていれば、中身は何でもいい
   - これこそが「ポリモーフィズム」の正体

3. **駅（API）の役割** (10分)
   - 乗客は電車の運転方法を知らなくても「駅」に行けば乗れる
   - 駅は「内部の複雑さ」を隠蔽するファサード

4. **落とし穴** (5分)
   - 乗り換えが多すぎると、目的地に着くのが遅くなる
   - 連結部分（API呼び出し）のオーバーヘッドで破綻する

## Key Takeaways

1. **連結器が合えば交換可能** — インターフェースの力
2. **駅は複雑さを隠す窓口** — ファサードパターン
3. **分割しすぎると遅くなる** — マイクロサービスの罠
