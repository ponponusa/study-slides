# エンジニアとしての考え方 Ver.1.0

**開催日**: 2026年1月13日
**テーマ**: なぜ、我々は「具体」と「抽象」を往復しなければならないのか？

## 概要

エンジニアの本質は「翻訳家」です。人間の曖昧な要求（具体）をコンピュータの厳密なロジック（抽象）へと変換する、この翻訳作業こそがエンジニアの価値です。

本勉強会では、「具体」と「抽象」という2つの世界を往復する重要性と、その往復速度を高めることで成長する方法について学びます。

## アジェンダ

1. **エンジニアの仕事の本質**
   - 「作業」と「翻訳」の違い
   - Human(具体・曖昧) と Computer(抽象・論理) の間の翻訳者

2. **「具体」とは何か？ (N=1)**
   - 具体例：ツナマヨおにぎり
   - わかりやすいが応用が効かない世界

3. **「抽象」とは何か？ (N=∞)**
   - 情報を捨てることで汎用性を得る
   - プログラミングの本質：変数、クラス、インターフェース

4. **罠①：具体の世界の住人**
   - 「コピペ・ボット」の末路
   - AI時代に淘汰される人材

5. **罠②：抽象の世界の住人**
   - 「アーキテクチャ宇宙飛行士」の末路
   - 現場で使えない理論の危険性

6. **解法：高速の往復運動**
   - 優秀なエンジニアのエレベーター
   - 螺旋の進化：認知領域の拡大

7. **One more thing: 本当の成長とは**
   - 「わかった気」の罠
   - Input vs Output: 汗をかく重要性
   - 「生徒」ではなく「審査員」になれ

## 重要な気づき

### 「わかった気」の罠
- サッカーの試合を見ても、サッカーは上手くならない
- Input だけでは成長しない。Output (言語化・実装) してこそ成長する

### 教わる側のリスク
- 講師を盲信するのは危険
- 「生徒」ではなく「審査員」になる
- 疑い、検証するプロセスこそが成長
